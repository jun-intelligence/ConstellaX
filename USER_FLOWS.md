# User Flows

## 1. Investor Demo Flow

1. User lands on `/`.
2. User understands ConstellaX as trusted infrastructure for the business of influence.
3. User opens `/demo`.
4. Demo walks through brief, recommendation, approval, contract/NDA, deliverables, flexible payment terms, and final report.
5. User opens campaign record for operational proof.

## 2. Campaign Workflow

1. Brand or agency captures client outreach and campaign brief.
2. Campaign record stores objective, budget, deliverables, rights needs, owners, and timeline.
3. AI intelligence recommends creators with rationale.
4. Manager and creator review terms, usage rights, deadlines, and fees.
5. Contract and NDA status are tracked.
6. Deliverables move through concept, first cut, final asset, and usage pack.
7. Approval queue shows owner, status, due date, and changes requested.
8. Invoice and payment timeline tracks deposit, milestone, usage fee, balance, overdue risk, and reminders.
9. Final report summarizes performance, audience, live posts, and renewal opportunities.

## 3. AI Manager Chat Studio

1. User opens `/dashboard/ai-manager/chat-studio`.
2. User selects role context: Creator, Manager, Agency, or Brand.
3. User reviews campaign context side panel.
4. User asks a strategy, creative, negotiation, workflow, or reporting question.
5. Mock AI returns structured advice and response cards.
6. User can save output to campaign, export PDF placeholder, send to manager placeholder, or add to deliverables placeholder.
7. Storyboard generator creates frame-by-frame creative direction.
8. Human confirmation remains required for messages, approvals, contracts, and payment terms.

## 4. Creator Without Representation

1. Creator opens dashboard.
2. AI Manager surfaces recommended next actions.
3. Creator asks Chat Studio for negotiation, creative, contract, or reporting guidance.
4. Creator reviews rate, usage rights, contract, and deadline guidance.
5. Creator tracks deliverables, approvals, invoices, and payments.
6. Creator uses report and audit trail for renewal negotiation.

## 5. Manager

1. Manager opens portfolio dashboard.
2. Manager reviews creator approvals, contracts, usage rights, payment states, and renewal opportunities.
3. Manager uses AI recommendations and Chat Studio to prepare negotiation points.
4. Manager confirms terms manually before sending or approving anything.

## 6. Brand

1. Brand reviews campaign needs and creator recommendations.
2. Brand compares fit, audience quality, engagement quality, ROI potential, and trust.
3. Brand approves shortlist, creative, contracts, and invoices.
4. Brand reviews final report and renewal recommendations.

## 7. Agency

1. Agency manages brand campaign operations.
2. Agency reviews creator shortlist, manager communications, approval queue, budgets, fees, contracts, invoices, and reporting.
3. Agency uses the workflow system to reduce coordination friction.
4. Agency keeps brands and managers aligned without losing the shared record.

## 8. Account And Settings

1. User can view mock login, signup, and forgot-password flows.
2. User opens settings.
3. User updates mock profile, role, notification, billing, security, and workspace preferences.
4. Future Supabase Auth should replace mock user state.

## 9. Live Post Tracking

1. Creator connects social accounts in future settings.
2. Campaign defines approved hashtags.
3. System tracks live posts that match connected accounts and campaign hashtags.
4. Reports display views, reach, ER, likes, comments, saves, shares, and audience breakdowns.
5. Current implementation is mock-only.
